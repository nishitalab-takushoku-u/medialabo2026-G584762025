本テーマについて

この講義では、情報工学の専門科目の基礎知識を応用して実践的な能力を習得することが目的になっており。 細かく分けると1,手順書を読み実験を行いレーポートを制作すること2,HTMLを使用したWebページ制作ができる3,課題に対する解決能力育成するなどがある

プロジェクト課題について

このページでは、プルダウンメニューから調べたい国を一つ選び項目のプルダウン（すべて、軽度、緯度、天気、最低気温、最高気温、湿度、風速、風向）の中から一つ選びその国のその項目と都市名を表示するページです。 まずHTMLにはプルダウンメニューを二つ置きその下にボタンを置く一つ目のプルダウンメニューには指定されたidをvalueに設定し二つ目のプルダウンメニューには情報が入っている場所をvalueに設定した。 jsには検索ボタンが押されたら指定された情報をサーバから入手し、その後ボタンの後ろにdiv要素を追加し検索結果というテキストと都市名を追加した後ul要素を追加する。 二つ目のプルダウンメニューで指定されている項目を表示するように表示したい項目の数だけli要素を追加し情報を表示するようになっている。加えて検索が二回以上押されたらdiv要素を削除するようになっている。

プロジェクト課題で作成したページ:project.html

数当てゲーム

このページでは、1~10の整数が自動で設定されるためその整数を三回以内に当てるとゆうものです。 まず変数として、回数を数えるkaisu,回数を表示する場所を示すn,予想した数を抜き出すa,結果を出力を示すrとおく。 実行されたら1~10の整数をランダムにkotaeという変数に代入しコンソールに出力する、加えて回数を初期値1として出力させる。 その後数値を入力されボタンが押されたら、kaisuを1増やしnを使って出力、yosoという変数にaを使って入力された数を代入し、yosoとkotaeを比べてその関係によってrを使いテキストを変更するというシステムになっています。

数当てゲームのページ:kazuate.html

各授業回の学習事項

第一回WWW と HTML

今回の講義では、WWW HTMLについて学んだ、URLにはプロトコル(https:)、サーバ、パスがあり、これらがインターネット上の膨大なデータからWebページを探すのに役立っている。 HTMLではレイアウトを変えるためにたくさんのタグがある。<p></p> ではタグ内の文が一つの要素となり <strong></strong> ではタグ内の文字が強調される <img src="〜〜"> では〜〜に画像ファイルを入力することで画像が表示される。画像ファイルの後ろに alt="〜〜" を入力すると閲覧者が読み上げソフトを使用していると〜〜内にある文字が読み上げられる。ul は番号なしのリストを作る、ol は番号付きのリストを作る（項目はliで作る）など学んだ。

演習の成果:演習1-4

第二回 CSS による Web ページの装飾

今回の講義では、主にCSSについて学んだhtmlにcssファイルをリンクさせるためには、 <link rel="stylesheet" href="[cssファイル名]"> を使用しcssファイルに変更したい内容を書き込んでいく。 編集するにはまず変更したいタグを決めて [タグ名]{} の中に記入していく。 coler:[色] で文字のいろの指定し、 text-shadow :で文字に影がつく。 border で文字の周りに線を敷くことができ、 border-[線を引く場所]-style:[線の種類] で線の引き方・線の種類（例：棒線solid、点線dashed など）を設定できる。 background では背景の色を指定でき linear-gradient でグラデーションをつけることができる。まわりのものから距離を作りたいときは、margin(線と前の要素との距離)、padding(線と文章の距離)を使うなど学んだ。

今回の講義では、DOMについて学んだ。DOMはページにボタンなどの機能を追加し動きをつけることができる。たとえば<button id="~~">~~</button>でページ内にボタンをつけることができIDやボタン上の文字を設定できる、<input type="text" name="shimei" id="shimei" size="10">ではテキストを入力できるテキストボックスを設置でき、nameで名前 idでID sizeで大きさを設定できる。 テキストボックスの場合はinput type="text"になるがtyoe="~~"の〜〜の部分を変更することで他にも様々なユーザーインターフェースを使うことができる（例 colorでカラービーカー、dateでカレンダー,search 検索キー　など）document.querySelector(~~~)では~~の要素を検索でき~~にp#messageと入力するとP要素のmessageを検索する。テキストボックス内のテキストを検索したいなら~~.textContentを使うなど学んだ。

今回の講義では前回に続きDOMについて学んだ。今回学んだのは主にプログラミングを使った要素の追加・削除・編集です。 具体的には。要素の追加にはdocument.createElement(〜〜)を使用し、要素1.insertAdjacentElement('beforeend', 要素2)で要素1の中に要素2を配置、要素1.insertAdjacentElement('afterend', 要素2)で要素1の要素の直後に要素2を配置するなどを使い追加する場所を指定する。 要素.setAttribute(属性名, 値)は要素に属性をつけることができ、要素.remove()は要素の削除ができる。これらを使用するとHTMLで要素を書き込む必要がなくなり、テキストを変える以外のこともプログラミングでできるようになることを学んだ。

第六回 プロジェクト課題を完成させよう

今回の講義では、情報通信処理について学んだ。通信はブラウザからサーバーへ通信する処理（リクエストする）とサーバからブラウザへ通信する処理（レスポンスを返す）の二つの処理を行う、 そしてリクエストを送ってレスポンスが帰ってくるまで待つことを同期通信といい、リクエストを送った後ブラウザの別の処理を実行しレスポンスの受け取りはイベント処理とすることを非同期通信という。 サーバからデータを受け取るにはaxiosを使用する場合HTMLに<script src="ajax-demo.js" defer></scrip>をheadに記入しjsに通信に成功した時、失敗した時、通信後の最後の処理を記入する必要があることを学んだ。

今回制作した課題の工夫点としては、いちいちキーボード入力しなくても済むようにプルダウンメニューを使用したことに加え、知りたい情報だけ調べられるように二つ目のプルダウンメニューを使用し検索ボタンが押されたら二つ目のプルダウンメニューで選ばれた内容だけ表示されるようにした点、そして複数の情報を表示するとき見やすくなるようul要素を使い箇条書きで表示されるようにした点です。 苦心点としては、HTMLやjavaScriptのプログラミングがjavaのプログラミングと違っており応用となるとなかなか手こずってしまい授業内に課題が終わらないことがあった。特にjavaScriptでのdocument.querySelector()による要素の検索をするときや、insertAdjacentElement()による要素の追加の時に()内に書く要素の書き方がなかなか掴めず時間がかかった点があった。他にもcssでのさまざまなページをデザインするためのコードがあるため、一つ一つを理解して使いこなすことができずpxなどを利用した指定通りの細かなデザインにするのが難しかった。 強調点としては、授業資料に記入してある大事そうな用語や、大事そうな概念、大事そうなコードなどは、パソコンのメモにコピーし、プロジェクト課題や演習で必要になったコードを資料から探し出さなくてもいいようにした点、cssによるページのデザインを見やすくする点、プロジェクト課題でのjavaScriptによるイベント処理において、初めて使う人が初見で使用できるような単純な仕様にした点などがあると思う。 感想としては、今回の講義を通してWebコンテンツの制作の基礎的な知識をたくさん習得ができるようになったが、それと同じくらいのWebコンテンツ作成難しさを知ることができた。いつも見ていたWebサイトがどのように作られているのか、どんなプログラムで組まれているのか知ることでWebページを制作する人たちの大変さ・こだわり・奥深さを直に感じることでWebを見るときの価値観が変わった。 最終講義で先生言っていたように現在はツールを使用してWebを簡単に作ることができるかもしれないが、この講義で学んだ内容は、ツールを使う上での注意点や応用方法を深く理解できるようになると思うのである程度覚えておきたい。しかし反省点としてプロジェクト課題に写真やイラストを使用しページの情報量を増やしたりページに動きをつけたりなどのもっと個性的なWebページを作りたかった。ので今後どんどん知識をつけてい期待と思った。
